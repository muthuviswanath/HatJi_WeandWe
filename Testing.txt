---------------Integration Testing----------------------
Testing the data flow between two or more modules (stories) is called as integration testing
How?
1. Dependency between the modules
2. Application should've passed the funcionality testing for the dependent modules

Incrementally: Forward & Backward
Non-Incrementally

When the actual module is not ready and instead of that the developers have included a dummy module which acts like a real module, it is called as stubs. 
------------ System Testing-------------------------------
1. Environment
		The resources that are required to peform the task
Development: The resources that are required to peform the task of creating, modifying the application
Testing: The resources that are required to peform the task of verifying and validating the apps functionality
Production: The resources that are required to peform the task of executing the build by the end users

Testing the application from the start module to the end module by going through all the other dependent modules. Alternatively testing the application E2E is called as system testing

1. The system should be functionally stable
2. There must be a MVP

For an application there must be one primary goal at least
There can be other multiple secondary goals

Build - The codes that are compiled, compressed and kept in .jar/.war/.ear/.zip/.exe format
which can be deployed in production

When there is a problem that is identified in the production and the production team is raising the bug
it is called defect. 

When the developers fix the defect it is called as hotfix

Re-release the build with the fixes and that is called as patch/updates

Re-release is called as Re-spin


To avoid re-release we have to make sure that the test environment is similar as the production environment


---------------------Compatibility Testing---------------------------------

Test the application in various platfroms or various browsers or various version of the same 
browser

If the application is developed using  a prior version and tested using the recent one, it is called
forward compatibility testing. The reverse is called backward compatibility testing

 The platform/browser is choosen based on the following:
 1. Requirements
 2. Popularity Index

 Why?
 To retain the customer base and to avoid bad names spreading in the market
